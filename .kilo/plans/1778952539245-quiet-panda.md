# Plan: Fix `next/headers` Import Error in WorkspaceBody.tsx

## Error
```
./components/custom/WorkspaceBody.tsx:10:1
You're importing a module that depends on "next/headers". This API is only available in
Server Components in the App Router, but you are using it in the Pages Router.
```

## Root Cause
`WorkspaceBody.tsx` is marked `"use client"` (Client Component) but imports
`{ cookies } from 'next/headers'` — a server-only API. This is Next.js 16
incompatible usage. Additionally, the component is declared `async`, which is
invalid for client components.

The `cookies()` import is **unused** — the token is already fetched via
`axios.get('/api/github/token')`, where `cookies()` is correctly used on the
server side in `app/api/github/token/route.ts`.

### `components/custom/WorkspaceBody.tsx`

2. **Remove `async`** from function declaration on line 12:

   - To: `function WorkspaceBody() {`
